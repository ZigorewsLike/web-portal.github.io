Подготовка конфигурации

Задача. Создайте конфигурацию со следующим набором прикладных объектов.

Классификаторы

Подсистемы
  1. Учет материалов
  2. Оказание услуг
  3. Предприятие
Перечисления
  1. Виды номенклатуры. Значения: Материал/Услуга
Справочники
  1. Клиенты (код 9, наименование 50)
  2. Сотрудники (код 9, наименование 50). Табличная часть Трудовая деятельность с реквизитами:
    1. Организация (строка 100),
    2. Начало работы (дата),
    3. Окончание работы (дата),
    4. Должность (строка 100)
  3. Номенклатура (код 9, наименование 100). Иерархия групп и элементов. Реквизит Вид номенклатуры
  4. Склады (код 9, наименование 100). Предопределенный элемент Основной

Документы

Для каждого реквизита документа и табличной части документа следует указать свойство Проверка заполнения.

  1. Приходная накладная. Отражает факт поступления материалов на склад. Реквизит Склад (ссылка на справочник). Табличная часть Материалы с реквизитами:
    1. Материал (ссылка на справочник)
    2. Количество (число 15,3)
    3. Цена (число 15,2)
    4. Сумма (число 15,2)
  2. Оказание услуги. Отражает факт оказание услуг и расхода материалов. Реквизиты:
    1. Склад (ссылка на справочник). Установить свойство ЗначениеЗаполнения,
    2. Клиент (ссылка на справочник)
    3. Мастер (ссылка на справочник)
    Табличная часть Перечень номенклатуры с реквизитами:
    1. Номенклатура (ссылка на справочник)
    2. Количество (число 15,3)
    3. Цена (число 15,2)
    4. Сумма (число 15,2)
Задания
  1. На формы документов требуется добавить пересчет суммы строки табличной части при изменении количества и цены.
  2. Дополнительно на формах нужно расположить реквизит, показывающий общую сумму всех строк табличной части документа.

Регистры

Для каждого измерения установите свойство Запрет незаполненных значений. В формы документов-регистраторов добавьте навигационные ссылки на соответствующие регистры для отслеживания совершенных движений

  1. Остатки материалов. Регистр накопления остатков. Измерение Материал (ссылка на справочник), Склад (ссылка на справочник). Ресурс Количество (число 15,3). Регистраторы: Приходная накладная и Оказание услуги.

    С помощью конструктора движений создайте движения по регистру Остатки материалов для двух регистраторов. Для движения документа Оказание услуги предусмотрите тот факт, что в регистре должны проводиться только материалы из табличной части этого документа.

  2. Цены. Регистр сведений. Измерение Номенклатура (ссылка на справочник). Ресурс Цена (число 15,2). Регистр периодический в пределах секунды и неподчиненный регистратору.

    Требуется реализовать функциональность по автоматическому заполнению цены услуги при выборе новой номенклатуры в документе Оказание услуги. Для отслеживания события для заполнения цены следует использовать обработчик события ПриИзменении поля Номенклатура табличной части документа.

  3. Стоимость материалов. Регистр накопления остатков. Измерение Номенклатура (ссылка на справочник). Ресурс Стоимость (число 15,2). Регистраторы: Приходная накладная и Оказание услуги.

    Самостоятельно добавьте в обработчик проведения документа Приходная накладная программный код для совершения движения по регистру Стоимость материалов. Ресурс Стоимость заполняется как сумма по конкретной позиции номенклатуры.

    Добавьте в табличную часть документа Оказание услуги новый реквизит Стоимость (число 15,2). Отобразите этот реквизит в форме документа. Самостоятельно добавьте в обработчик проведения документа Оказание услуги код для совершения движения по регистру Стоимость материалов. Ресурс Стоимость заполняется как произведение количества и стоимости по конкретной позиции номенклатуры.

  4. Продажи. Регистр накопления оборотов. Измерение Номенклатура (ссылка на справочник), Клиент (ссылка на справочник), Мастер (ссылка на справочник) . Ресурсы Количество (число 15,3), Выручка (число 15,2), Стоимость (число 15,2). Регистратор Оказание услуги.

    5. Самостоятельно добавьте в обработчик проведения документа Оказание услуги код для совершения движения по регистру Продажи. Ресурс Выручка заполняется как сумма по конкретной позиции номенклатуры. Ресурс Стоимость заполняется как произведение количества и стоимости по конкретной позиции номенклатуры.

    Обратите внимание, что в регистре Продажи в качестве значения измерения может выступать как материал, так и услуга.

Задания
  1. На форму документа Оказание услуги добавьте подстановку актуальной цены в соответствующую колонку табличной части при выборе материала или услуги.
  2. На форму элемента справочника Номенклатура добавьте реквизиты "Цена", "Дата" и кнопку "Обновить". При открытии формы реквизит "Цена" заполняется данными из регистра Цены, реквизит "Дата" указывает дату начала действия заданной цены. Пользователь может изменить цену номенклатуры в помощью кнопки "Обновить" указав новую цену и новую дату. Перед записью новой цены в регистр требуется попросить пользователя подтвердить операцию.
  3. На форму списка справочника Номенклатура добавьте два реквизита, которые для выбранной номенклатуры указывать ее остаток и цену на текущую дату.

Оптимизация работы регистров и введение стоимостного учета

Получение вида для всей номенклатуры за одно обращение к БД

Необходимо сделать более эффективным проведение документа Оказание услуги. Для этого требуется получать вид для всей номенклатуры за одно обращение к базе данных. Получение информации о виде номенклатуры из текущего документа возможно через запрос к базе данных с отбором по документу (он еще не проведен, но уже записан в БД). Дополнительно следует получить количество номенклатуры по позициям, сумму и стоимость.

Ведение стоимостного учета

До сих пор стоимость расходуемых материалов пользователи вписывали в документ Оказание услуги вручную, при его создании. Теперь же компания будет определять стоимость номенклатуры по среднему: для каждой номенклатуры делить ее общую, суммарную стоимость на то количество этой номенклатуры, которое имеется, и таким образом получать среднюю стоимость одной единицы этой номенклатуры.

Чтобы выполнить такой расчет для каждой номенклатуры из табличной части понадобится: ее стоимость, хранящаяся в регистре СтоимостьМатериалов и общее ее количество на всех складах, хранящееся в регистре ОстаткиМатериалов.

Контроль остатков пр оперативном проведении

При оперативном проведении документа Оказание услуги требуется проверять доступные остатки номенклатуры и если их не хватает, то останавливать проведение документа.
